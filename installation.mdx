CandyShell is a lightweight Wayland desktop shell built with GTK4, Adwaita, and Lua. Follow these steps to install all required dependencies and get CandyShell running on your system.

## System Requirements

CandyShell requires a Wayland compositor (such as Sway) and several core dependencies:

- **LuaJIT** - Just-In-Time Compiler for Lua
- **GTK4** - The GIMP Toolkit version 4
- **Adwaita (libadwaita)** - GNOME's adaptive UI library
- **Gtk4-Layer-Shell** - Library for layer shell protocol
- **lgi** - Lua bindings for GObject introspection
- **matugen** - Material You color generation tool (for dynamic theming)

<Note>
CandyShell is designed specifically for **Wayland** and requires a compositor that supports the layer-shell protocol, such as Sway or Hyprland.
</Note>

## Installation by Distribution

<Steps>

<Step title="Install core dependencies">

Choose your distribution:

<CodeGroup>

```bash Arch Linux
# Install core packages
sudo pacman -S luajit gtk4 libadwaita gtk4-layer-shell lua-lgi

# Install matugen for dynamic theming
yay -S matugen-bin
```

```bash Fedora
# Install core packages
sudo dnf install luajit gtk4 libadwaita gtk4-layer-shell lua-lgi

# Install matugen from cargo
cargo install matugen
```

```bash Ubuntu/Debian
# Install core packages
sudo apt install luajit libgtk-4-dev libadwaita-1-dev gtk4-layer-shell lua-lgi

# Install matugen from cargo
cargo install matugen
```

</CodeGroup>

<Note>
The `lgi` package provides Lua bindings for GObject Introspection, which allows CandyShell to access GTK4 libraries from Lua code.
</Note>

</Step>

<Step title="Clone the repository">

Clone CandyShell from the source repository:

```bash
git clone https://github.com/your-username/candyshell.git
cd candyshell
```

</Step>

<Step title="Verify dependencies">

Ensure all dependencies are correctly installed by checking their versions:

```bash
# Check LuaJIT
luajit -v

# Check GTK4 (should output GTK version)
pkg-config --modversion gtk4

# Check if lgi can load GTK4
luajit -e "require('lgi').require('Gtk', '4.0'); print('GTK4 bindings OK')"

# Check matugen
matugen --version
```

All commands should complete without errors.

</Step>

<Step title="Prepare resource files">

CandyShell expects certain resource files to be present:

```bash
# Ensure the res directory exists with required files
ls res/
```

Required files:
- `res/wall.png` - Default wallpaper (or any image file)
- `res/icon.jpg` - Profile icon
- `res/pfp.jpg` - Profile picture for dashboard
- `res/no_music.png` - Placeholder for music widget

<Warning>
If these files are missing, CandyShell may fail to start or display placeholder errors. You can replace these with your own images.
</Warning>

</Step>

</Steps>

## Verifying Installation

Before running CandyShell, verify that the main executable can load all required modules:

```lua main.lua (excerpt)
-- These are the core dependencies loaded by CandyShell
local ffi = require("ffi")
ffi.C = ffi.load("gtk4-layer-shell")

local lgi = require("lgi")
local Gtk = lgi.require("Gtk", "4.0")
local LayerShell = lgi.require("Gtk4LayerShell", "1.0")
local GLib = lgi.require("GLib")
local Gio = lgi.Gio
local Adw = lgi.require("Adw")
```

Test loading these modules:

```bash
luajit -e "require('lgi').require('Gtk4LayerShell', '1.0'); print('Layer Shell OK')"
luajit -e "require('lgi').require('Adw'); print('Adwaita OK')"
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: module 'lgi' not found">
    The Lua GObject Introspection bindings are not installed. Install `lua-lgi` for your distribution:
    
    ```bash
    # Arch
    sudo pacman -S lua-lgi
    
    # Or build from source
    git clone https://github.com/pavouk/lgi.git
    cd lgi
    make
    sudo make install
    ```
  </Accordion>
  
  <Accordion title="Error: cannot load 'gtk4-layer-shell'">
    The GTK4 Layer Shell library is missing. Install it:
    
    ```bash
    # Arch
    sudo pacman -S gtk4-layer-shell
    
    # Fedora
    sudo dnf install gtk4-layer-shell
    ```
  </Accordion>
  
  <Accordion title="GTK4 version mismatch">
    Ensure you have GTK4 (not GTK3) installed:
    
    ```bash
    pkg-config --modversion gtk4
    ```
    
    The version should be 4.0 or higher.
  </Accordion>
  
  <Accordion title="matugen command not found">
    Install matugen for dynamic theming:
    
    ```bash
    # Via AUR (Arch)
    yay -S matugen-bin
    
    # Via Cargo (all distributions)
    cargo install matugen
    ```
  </Accordion>
</AccordionGroup>
