## Widget System Overview

CandyShell's widget system is built on **GTK4** using Lua, providing a flexible architecture for creating self-contained, stateful UI components. Widgets manage their own lifecycle, updates, and styling.

<Note>
  All widgets in CandyShell follow a factory function pattern that returns configured GTK widgets with embedded update logic.
</Note>

## Widget Lifecycle

### Factory Function Pattern

Widgets are created using factory functions that:

1. Create GTK widget instances
2. Configure appearance and behavior
3. Set up update timers
4. Return the configured widget

```lua
local function create_widget_name()
  -- 1. Create widget
  local widget = Gtk.Label.new("Initial Value")
  
  -- 2. Configure
  widget:add_css_class("widget-style")
  widget.halign = Gtk.Align.CENTER
  
  -- 3. Set up updates
  local function update_widget()
    widget:set_text(get_current_value())
    return true  -- Return true to keep timer running
  end
  
  GLib.timeout_add_seconds(GLib.PRIORITY_DEFAULT, 5, update_widget)
  update_widget()  -- Initial call
  
  -- 4. Return widget
  return widget
end
```

<Accordion title="Why Factory Functions?">
  Factory functions provide several benefits:
  
  - **Encapsulation** - State is local to the function closure
  - **Reusability** - Call the factory multiple times for multiple instances
  - **Self-contained** - Each widget manages its own lifecycle
  - **Garbage Collection** - Closures are cleaned up when widget is destroyed
</Accordion>

### Simple Label Widget

The time label demonstrates the simplest widget pattern:

```lua

local function create_time_label()
  local time_label = Gtk.Label.new(os.date("%H:%M"))
  time_label:add_css_class("sys-status-text")

  local function update_time()

    return true

```

  </Step>

The battery label shows data fetching and icon selection:

```lua
local function create_battery_label()
  local battery_label = Gtk.Label.new("")
  battery_label:set_use_markup(true)
  battery_label:add_css_class("sys-status-text")

  -- Icon map for different battery levels
  local fa_icons = {
    charging = "󰂄", full = "󰁹", battery_90 = "󰂂",
    battery_80 = "󰂁", battery_70 = "󰂀", battery_60 = "󰁿",
    battery_50 = "󰁾", battery_40 = "󰁽", battery_30 = "󰁼",
    battery_20 = "󰁻", battery_10 = "󰁺", battery_0 = "󰂎"
  }

  local function update_battery()
    local percentage = SystemInfo.get_battery_percentage() or 0
    local state = SystemInfo.get_battery_state() or 0
    local is_charging = (state == 1)
    local icon = ""

    -- Icon selection logic based on state and percentage
    if is_charging then
      icon = fa_icons.charging
    elseif percentage >= 95 then icon = fa_icons.full
    elseif percentage >= 85 then icon = fa_icons.battery_90
    -- ... more conditions
    end

    battery_label:set_markup(string.format(
      '<span>%s %d%%</span>',
      icon, math.floor(percentage)
    ))
    return true
  end

  GLib.timeout_add_seconds(GLib.PRIORITY_DEFAULT, 30, update_battery)
  update_battery()
  return battery_label
end
```

<Tabs>
  <Tab title="Icon Selection">

  </Tab>
  <Tab title="Markup Usage">
    `set_use_markup(true)` enables Pango markup for rich text formatting. The `<span>` tag allows inline styling.
  </Tab>
  <Tab title="Update Frequency">
    Battery updates every 30 seconds to balance responsiveness with CPU usage. Frequent battery queries can impact performance.
  </Tab>
</Tabs>

### Custom Drawing with Cairo

The progress indicator uses Cairo for custom drawing:

```lua
-- dashboard.lua:226-273
local function draw_progress_wave(area, cr, width, height)
  local y_center = height / 2
  local active_length = width * current_progress
  local line_height = 3.0

  -- Draw inactive portion (gray)
  if active_length < width then
    cr:set_source_rgba(0.5, 0.5, 0.5, 0.25)
    cr:set_line_width(line_height)
    cr:move_to(active_length, y_center)
    cr:line_to(width, y_center)
    cr:stroke()
  end

  -- Draw active portion with wave animation
  cr:set_source_rgb(wave_color.r, wave_color.g, wave_color.b)
  cr:set_line_width(line_height)
  
  if current_progress > 0 then
    local wave_amplitude = is_playing_state and 3.5 or 0.0
    local wave_frequency = 6.0
    local wave_speed = 6.0
    
    cr:move_to(0, y_center)
    for i = 1, num_segments do
      local x_pos = (i / num_segments) * active_length
      local wave_offset = math.sin(
        (x_pos / width) * math.pi * wave_frequency * 2 + 
        animation_time * wave_speed
      )
      local y_pos = y_center + (wave_offset * wave_amplitude)
      cr:line_to(x_pos, y_pos)
    end
    cr:stroke()
  end
end

progress_area:set_draw_func(draw_progress_wave, nil, nil)
```

<Accordion title="Cairo Drawing Concepts">
  Cairo is a 2D graphics library that provides:
  
  - **Paths** - Define shapes with move_to, line_to, arc, etc.
  - **Strokes** - Draw path outlines with set_line_width, set_source_rgb
  - **Fills** - Fill path interiors
  - **Transformations** - Rotate, scale, translate
  - **Compositing** - Alpha blending, masks
  
  The `set_draw_func` callback is called whenever the widget needs repainting.
</Accordion>

## Calendar Widget

The calendar demonstrates grid layouts and date calculations:

```lua
-- calendario.lua:38-95
function M.crear_calendario(mes, anio)
  local calendario_box = Gtk.Box.new(Gtk.Orientation.VERTICAL, 10)
  calendario_box:add_css_class("box_dashboard")

  -- Header
  local cal_mes = Gtk.Label.new(M.wmonths[mes] .. ", " .. anio)
  cal_mes:add_css_class("cal-header")

  -- Grid for days
  local cal_dias = Gtk.Grid.new()
  
  -- Calculate days in month and starting day
  local total_dias = M.dias_en_mes(mes, anio)
  local dia_inicio = M.obtener_dia_inicio(mes, anio)
  local dia_actual = (mes == f.month and anio == f.year) and f.day or nil

  -- Fill grid
  local fila = 2
  local dia_numero = 1

  while dia_numero <= total_dias do
    for c = 1, 7 do
      if (fila == 2 and c < dia_inicio) or dia_numero > total_dias then
        -- Empty cell
        local vacio = Gtk.Label.new("")
        cal_dias:attach(vacio, c, fila, 1, 1)
      else
        -- Day cell
        local cal_dia = Gtk.Label.new(tostring(dia_numero))
        cal_dia:add_css_class("cal-day")
        
        if dia_numero == dia_actual then
          cal_dia:add_css_class("cal-today")
        end
        
        cal_dias:attach(cal_dia, c, fila, 1, 1)
        dia_numero = dia_numero + 1
      end
    end
    fila = fila + 1
  end

  calendario_box:append(cal_mes)
  calendario_box:append(cal_dias)
  return calendario_box
end
```

<Tabs>
  <Tab title="Grid Layout">
    `Gtk.Grid` positions widgets in rows and columns using `attach(widget, col, row, width, height)`. Perfect for calendar layouts.
  </Tab>
  <Tab title="Date Calculations">
    Lua's `os.date()` and `os.time()` handle date arithmetic. The widget calculates leap years, month lengths, and starting days.
  </Tab>
  <Tab title="Conditional Styling">
    The current day gets the `cal-today` CSS class for highlighting, showing dynamic style application.
  </Tab>
</Tabs>

### Auto-Updating Calendar

```lua
-- calendario.lua:97-134
function M.updatable_calendar()
  local calendar_container = Gtk.Box.new(Gtk.Orientation.VERTICAL, 0)
  local current_calendar = nil
    
  local function update_calendar()
    local f = os.date("*t")
    local new_calendar = M.crear_calendario(f.month, f.year)
        
    -- Replace current calendar
    if current_calendar then
      calendar_container:remove(current_calendar)
      current_calendar = nil
      collectgarbage("step") 
    end
        
    calendar_container:append(new_calendar)
    current_calendar = new_calendar
    return false
  end
    
  -- Schedule midnight updates
  GLib.timeout_add_seconds(GLib.PRIORITY_DEFAULT, 60, function()
    update_calendar()
    return true
  end)
    
  current_calendar = M.crear_calendario()
  calendar_container:append(current_calendar)
    
  return calendar_container
end
```

<Warning>
  Always remove old widgets from containers before appending new ones, and call `collectgarbage()` to free memory.
</Warning>

### GTK Widget Properties

Common widget properties set in Lua:

```lua
-- Alignment
widget.halign = Gtk.Align.CENTER  -- START, END, FILL
widget.valign = Gtk.Align.CENTER

-- Expansion
widget.hexpand = true  -- Fill available horizontal space
widget.vexpand = true

-- Size
widget:set_size_request(width, height)  -- Minimum size

-- Margins
widget.margin_start = 10
widget.margin_end = 10
widget.margin_top = 5
widget.margin_bottom = 5
```

### Inline Styling

For dynamic styles, use CSS providers:


## Update Patterns

### Timer-Based Updates

```lua
-- Update every N seconds
GLib.timeout_add_seconds(GLib.PRIORITY_DEFAULT, interval, function()
  update_data()
  return true  -- Keep timer running
end)
```

### One-Shot Updates

```lua
-- Run once after delay
GLib.timeout_add(GLib.PRIORITY_DEFAULT, delay_ms, function()
  do_something()
  return false  -- Stop timer
end)
```

### Async Callbacks

```lua
-- dashboard.lua:70-75
w.update(10, 10.48, -66.90, function(data)
  GLib.idle_add(GLib.PRIORITY_DEFAULT_IDLE, function()
    refresh_ui()
    return false
  end)
end)
```

<Note>
  Use `GLib.idle_add` to schedule UI updates from async callbacks. This ensures updates run on the main thread.
</Note>

## Creating Custom Widgets

Here's a template for creating new widgets:

```lua
local function create_my_widget()
  -- 1. State variables
  local state = {}
  
  -- 2. Create GTK widget
  local widget = Gtk.Box.new(Gtk.Orientation.VERTICAL, 5)
  widget:add_css_class("my-widget")
  
  -- 3. Create children
  local label = Gtk.Label.new("Initial")
  widget:append(label)
  
  -- 4. Update function
  local function update()
    local data = fetch_data()
    label:set_text(data)
    return true
  end
  
  -- 5. Set up timers
  GLib.timeout_add_seconds(GLib.PRIORITY_DEFAULT, 5, update)
  update()  -- Initial update
  
  -- 6. Event handlers
  local click = Gtk.GestureClick.new()
  click.on_pressed = function()
    handle_click()
  end
  widget:add_controller(click)
  
  -- 7. Return widget
  return widget
end
```
