## Core components

CandyShell consists of three main visual layers: the background wallpaper window, the pull-down dashboard, and the bottom application bar.

### Wallpaper layer

The wallpaper sits on the `BACKGROUND` layer and responds to mouse events:

<CodeGroup>
```lua main.lua
local wallpaper = Gtk.Picture.new_for_filename(current_wp_path)
wallpaper.content_fit = Gtk.ContentFit.COVER

local click_controller = Gtk.GestureClick.new()
click_controller:set_button(0) -- Listen to all buttons

function click_controller:on_pressed(n_press, x, y)
  local btn = self:get_current_button()
  if btn == 1 then
    -- Left click: open file picker for new wallpaper
  elseif btn == 3 then
    -- Right click: toggle light/dark theme
  end
end
```
</CodeGroup>

The wallpaper automatically triggers theme regeneration when changed, extracting color palettes for the entire interface.

### Dashboard overlay

The dashboard appears when you hover near the top edge of the screen and auto-hides when you move away.

```lua dashboard.lua
local motion_controller = Gtk.EventControllerMotion.new()

function motion_controller:on_enter()
  revealer.reveal_child = true
  if conn then
    sway.run_command(conn, "gaps top all set 310")
  end
end

function motion_controller:on_leave()
  revealer.reveal_child = false
  if conn then
    sway.run_command(conn, "gaps top all set 10")
  end
end
```

This integration with Sway's IPC creates space for the dashboard by adjusting window gaps dynamically.

### Bottom bar

The bottom bar uses the `TOP` layer with auto-exclusive zone to reserve screen space:

```lua main.lua
LayerShell.init_for_window(bar_win)
LayerShell.set_layer(bar_win, LayerShell.Layer.TOP)
LayerShell.set_anchor(bar_win, LayerShell.Edge.BOTTOM, true)
LayerShell.auto_exclusive_zone_enable(bar_win)
```

It contains three sections: launcher, pinned apps, and system status.

function M.get_album_colors()
  local pixbuf = M.get_album_art()
  if not pixbuf then 
    return "#353535", "#ffffff" -- Default colors
  end
  
  -- Scale to 3x3 pixels to get average color
  local one_pixel = pixbuf:scale_simple(3, 3, 'BILINEAR')
  local bytes = one_pixel:read_pixel_bytes()
  local data = bytes:get_data()
  
  local r = string.byte(data, 1)
  local g = string.byte(data, 2)
  local b = string.byte(data, 3)
  
  local bg_hex = string.format("#%02x%02x%02x", r, g, b)
  local fg_hex = get_contrasting_text_color(r, g, b)
  
  return bg_hex, fg_hex
end

### System monitors

Real-time system information using libgtop:

```lua libs/system_i.lua
function M.get_cpu_used()
  local cpu = GTop.glibtop_cpu()
  GTop.glibtop_get_cpu(cpu)
  
  local total_diff = cpu.total - last_total
  local idle_diff = cpu.idle - last_idle
  
  if total_diff > 0 then
    local usage = 100 * (1 - (idle_diff / total_diff))
    return math.min(100, math.max(0, usage))
  end
  return 0
end

function M.get_mem_info()
  local mem = GTop.glibtop_mem()
  GTop.glibtop_get_mem(mem)
  
  local real_used = mem.used - mem.buffer - mem.cached
  
  return {
    ram = {
      real_used = to_mb(real_used),
      percent_real_used = (real_used / mem.total) * 100
    }
  }
end
```

Monitored resources:
- **CPU usage** - Delta calculation for accurate percentage
- **Memory usage** - Real usage excluding buffers/cache
- **Disk usage** - Filesystem utilization for specified mount point
- **Battery status** - Percentage and charging state with icon indicators

## Theme system

CandyShell's theme system bridges wallpaper colors with GTK CSS:

### Color extraction

```lua libs/theme.lua
function M.apply(image_path, selected_mode)
  -- Run matugen to extract color palette
  local cmd = string.format("matugen image '%s' -j hex", image_path)
  local result = io.popen(cmd):read("*a")
  
  -- Parse JSON response
  local root = parser:get_root():get_object()
  local colors_obj = root:get_object_member("colors")
  local palette = colors_obj:get_object_member(selected_mode)
  
  -- Generate CSS variables
  local css_lines = {}
  for _, name in ipairs(palette:get_members()) do
    local value = palette:get_string_member(name)
    table.insert(css_lines, 
      string.format("@define-color %s %s;", name, value))
  end
  
  -- Apply to GTK
  css_provider:load_from_data(final_css, #final_css)
end
```

### Persistent settings

Theme preferences are saved to `~/.config/candy-shell/config.json`:

```json
{
  "wallpaper": "/path/to/image.png",
  "theme_mode": "dark"
}
```

The configuration is loaded on startup to restore your previous theme.

### CSS integration

Extracted colors are used throughout `custom.css` to style all components:

- Dashboard background and cards
- Button hover states
- Text colors based on background luminance
- Accent colors for interactive elements
- Smooth transitions when themes change

## Sway IPC integration

Direct communication with the Sway compositor:

```lua libs/sway.lua
function M.send_raw(conn, command, msg_type)
  -- Construct IPC header: magic + length + type
  local header_struct = ffi.new("sway_ipc_header_t")
  header_struct.length = #command
  header_struct.type = msg_type
  
  -- Send magic, header, payload
  output:write(IPC_MAGIC, nil)
  output:write(ffi.string(header_bytes, ffi.sizeof(header_struct)), nil)
  output:write(command, nil)
  
  -- Read response
  local response = input:read_bytes(resp_length, nil)
  return response:get_data()
end
```

Available IPC commands:
- `get_workspaces` - Query workspace information
- `get_outputs` - Monitor configuration
- `get_tree` - Window tree structure
- `run_command` - Execute arbitrary Sway commands
- `exec` - Launch applications
- `workspace` - Switch workspaces
- `focus` - Change window focus

## Performance optimizations

### Memory management

Periodic garbage collection prevents memory leaks:

```lua main.lua
GLib.timeout_add_seconds(GLib.PRIORITY_LOW, 30, function()
  collectgarbage("collect")
  return true
end)
```

Widgets explicitly nil out large objects after use:

```lua
pixbuf = nil
one_pixel = nil
bytes = nil
collectgarbage("step")
```

### Update intervals

Each component uses appropriate refresh rates:

- **Time display**: 1 second
- **System monitors**: 2 seconds  
- **Battery status**: 30 seconds
- **WiFi strength**: 15 seconds
- **Music player**: 1 second (metadata), 16ms (animation)
- **Weather**: 5 minutes
- **Calendar**: 60 seconds

### Caching strategies

Frequently accessed data is cached:

```lua libs/system_i.lua
local battery_cache = nil
local battery_cache_time = 0

function M.get_battery_info()
  local current_time = os.time()
  if battery_cache and (current_time - battery_cache_time) < 5 then
    return battery_cache
  end
  -- Fetch new data
end
```

<Note>
  Battery information is cached for 5 seconds to avoid excessive UPower queries.
</Note>
