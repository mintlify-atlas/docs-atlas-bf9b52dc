CandyShell is built with Lua and GTK4 using the LGI (Lua GObject Introspection) library. This reference covers the core APIs, system libraries, and common patterns used throughout the codebase.

## Core Dependencies

CandyShell relies on several libraries loaded via LGI:

```lua
local lgi = require("lgi")
local Gtk = lgi.require("Gtk", "4.0")
local Gdk = lgi.require("Gdk", "4.0")
local GLib = lgi.require("GLib")
local Gio = lgi.Gio
local Adw = lgi.require("Adw")  -- Libadwaita
local LayerShell = lgi.require("Gtk4LayerShell", "1.0")
```

### GTK4 Layer Shell

CandyShell uses GTK4 Layer Shell to integrate with Wayland compositors:

```lua
local ffi = require("ffi")
ffi.C = ffi.load("gtk4-layer-shell")

-- Initialize layer shell for a window
LayerShell.init_for_window(window)

-- Set layer (BACKGROUND, BOTTOM, TOP, OVERLAY)
LayerShell.set_layer(window, LayerShell.Layer.BACKGROUND)

-- Anchor to screen edges
LayerShell.set_anchor(window, LayerShell.Edge.LEFT, true)
LayerShell.set_anchor(window, LayerShell.Edge.TOP, true)

-- Set exclusive zone (reserves space)
LayerShell.set_exclusive_zone(window, 50)
LayerShell.auto_exclusive_zone_enable(window)

-- Set margins
LayerShell.set_margin(window, LayerShell.Edge.TOP, 10)
```

## System Information API

`libs/system_i.lua` provides system monitoring functions using GTop and UPower.

### Memory Information

```lua
local SystemInfo = require('libs.system_i')

local mem_info = SystemInfo.get_mem_info()
-- Returns:
-- {
--   ram = {
--     total = 16384,           -- MB
--     used = 8192,             -- MB
--     free = 8192,             -- MB
--     real_used = 6144,        -- MB (excluding buffers/cache)
--     percent_real_used = 37.5,
--     percent_used = 50.0
--   },
--   swap = {
--     total = 8192,            -- MB
--     used = 0,                -- MB
--     free = 8192,             -- MB
--     percent_used = 0.0
--   }
-- }

print(string.format("RAM: %.1f%%", mem_info.ram.percent_real_used))
```

### CPU Usage

```lua
local cpu_percent = SystemInfo.get_cpu_used()
-- Returns: 0-100 (percentage)

print(string.format("CPU: %.1f%%", cpu_percent))
```

<Warning>
The first call to `get_cpu_used()` always returns 0 because it needs a baseline measurement. Subsequent calls will return accurate values.
</Warning>

### Disk Usage

```lua
local disk_percent = SystemInfo.get_disk_usage('/')
-- Returns: 0-100 (percentage used)

local home_percent = SystemInfo.get_disk_usage('/home')
```

### Battery Information

```lua
-- Get battery percentage
local percentage = SystemInfo.get_battery_percentage()
-- Returns: 0-100

-- Get battery state
local state = SystemInfo.get_battery_state()
-- Returns:
--   0 = Unknown
--   1 = Charging
--   2 = Discharging
--   3 = Empty
--   4 = Fully charged
--   5 = Pending charge
--   6 = Pending discharge

local is_charging = (state == 1)

-- Get detailed battery info
local info = SystemInfo.get_battery_info()
-- Returns:
-- {
--   percentage = 75.5,
--   state = 2,
--   time_empty = 7200,    -- seconds until empty
--   time_full = 0         -- seconds until full (0 when discharging)
-- }
```

## Network Manager API

`libs/nm.lua` provides network status information.

### WiFi Status

```lua
local NetworkManager = require('libs.nm')

-- Check if WiFi is connected
local connected = NetworkManager.is_wifi_connected()
-- Returns: true/false

-- Get signal strength
local strength = NetworkManager.get_wifi_strength()
-- Returns: 0-100 (percentage)

if connected then
  if strength >= 75 then
    print("Excellent signal")
  elseif strength >= 50 then
    print("Good signal")
  elseif strength >= 25 then
    print("Fair signal")
  else
    print("Weak signal")
  end
else
  print("Not connected")
end
```

## Music Player Control API

`libs/music.lua` provides MPRIS integration using Playerctl.

### Player Information

```lua
local music = require('libs.music')

-- Get all player info
local info = music.player_info()
-- Returns:
-- {
--   artist = "Artist Name",
--   title = "Song Title",
--   album = "Album Name",
--   artUrl = "file:///path/to/art.jpg",
--   length = "180000000",  -- microseconds
--   status = "PLAYING"     -- PLAYING, PAUSED, STOPPED, EXITED
-- }

-- Individual accessors
local artist = music.get_artist()
local title = music.get_title()
local album = music.get_album()
```

### Playback Control

```lua
-- Toggle play/pause
music.play_pause()

-- Skip to next track
music.next()

-- Go to previous track
music.previous()
```

### Track Position

```lua
-- Get current position (seconds)
local position = music.get_position()

-- Get track length (seconds)
local length = music.length()

-- Get formatted time
local formatted = music.length_min_sec()
-- Returns: "3:45"

-- Calculate progress percentage
local progress = (length > 0) and (position / length) or 0.0
```

### Album Art

```lua
-- Get album art as GdkPixbuf
local pixbuf = music.get_album_art()
-- Returns: GdkPixbuf or nil

if pixbuf then
  local image = Gtk.Image.new_from_pixbuf(pixbuf)
  image:set_pixel_size(100)
else
  -- Use fallback image
  local image = Gtk.Image.new_from_file('res/no_music.png')
end

-- Get dominant colors from album art
local bg_color, text_color = music.get_album_colors()
-- Returns: "#FF5733", "#FFFFFF"
-- Automatically calculates contrasting text color
```

## Weather API

`libs/weather.lua` provides weather information using GWeather (libgweather).

### Fetching Weather Data

```lua
local weather = require('libs.weather')

-- Update weather data (async)
weather.update(timeout, latitude, longitude, function(data)
  print("Location: " .. data.state .. ", " .. data.country)
  print("Temperature: " .. data.temp)
  print("Conditions: " .. data.descrip)
  print("Wind: " .. data.wind)
  print("Humidity: " .. data.humi)
  print("Emoji: " .. data.emoji)
end)

-- Example coordinates: Caracas, Venezuela (10.48, -66.90)
weather.update(10, 10.48, -66.90, callback)
```

### Weather Data Structure

```lua
-- Access cached weather data
if weather.datos.load then
  print(weather.datos.state)      -- City name
  print(weather.datos.country)    -- Country name
  print(weather.datos.temp)       -- Temperature string (e.g., "24°C")
  print(weather.datos.descrip)    -- Conditions (e.g., "Partly cloudy")
  print(weather.datos.emoji)      -- Weather emoji (e.g., "󰖕")
  print(weather.datos.wind)       -- Wind speed
  print(weather.datos.humi)       -- Humidity
  print(weather.datos.icon)       -- Icon name (e.g., "weather-few-clouds")
else
  print("Weather data not loaded yet")
end
```

<Warning>
`weather.update()` blocks the main loop until completion or timeout. Always set a reasonable timeout (5-10 seconds) to avoid freezing the UI.
</Warning>

## Theme Management API

`libs/theme.lua` handles dynamic theming using matugen for color extraction.

### Applying Themes

```lua
local Theme = require('libs.theme')

-- Apply theme from wallpaper
Theme.apply("/path/to/wallpaper.jpg", "dark")
-- Modes: "dark" or "light"

-- Save theme settings
Theme.save_settings("/path/to/wallpaper.jpg", "dark")
-- Saves to ~/.config/candy-shell/config.json

-- Load saved settings
local wallpaper_path, theme_mode = Theme.get_saved_settings()
-- Returns: "/path/to/wallpaper.jpg", "dark"
-- Defaults: "res/wall.png", "dark" if no settings saved
```

### How Theme Generation Works

<Steps>
  <Step title="Extract colors">
    Matugen analyzes the wallpaper and generates a Material Design 3 color palette:
    ```bash
    matugen image '/path/to/wallpaper.jpg' -j hex
    ```
  </Step>
  
  <Step title="Parse JSON output">
    The JSON response contains light and dark mode color schemes:
    ```json
    {
      "colors": {
        "dark": {
          "primary": "#BB86FC",
          "on_primary": "#000000",
          "surface": "#121212",
          ...
        },
        "light": { ... }
      }
    }
    ```
  </Step>
  
  <Step title="Generate CSS">
    Colors are converted to CSS variables:
    ```css
    @define-color primary #BB86FC;
    @define-color on_primary #000000;
    @define-color surface #121212;
    ```
  </Step>
  
  <Step title="Apply to GTK">
    The CSS is loaded via GtkCssProvider, updating all styled widgets instantly.
  </Step>
</Steps>

## Application Launcher API

`libs/gapps.lua` provides access to installed desktop applications.

### Getting Applications

```lua
local GApps = require('libs.gapps')

-- Get all installed applications
local all_apps = GApps.get_all_apps()
-- Returns: table indexed by desktop file ID

-- Access specific app
local app_data = all_apps["firefox.desktop"]
if app_data then
  print(app_data.name)        -- Display name
  print(app_data.description) -- App description
  print(app_data.icon)        -- GIcon object
  
  -- Launch the application
  app_data.launch()
end

-- Iterate over all apps
for app_id, app_data in pairs(all_apps) do
  print(app_id .. ": " .. app_data.name)
end
```

### Creating App Buttons

Example from `main.lua`:

```lua main.lua (lines 129-149)
local function create_app_button(app_id)
  local all_apps = GApps.get_all_apps()
  local app_data = all_apps[app_id]

  if not app_data then return nil end

  local button = Gtk.Button.new()
  button:add_css_class("apps-box")

  local image = Gtk.Image.new_from_gicon(app_data.icon)
  image:set_pixel_size(32)
  
  button:set_child(image)
  button:set_tooltip_text(app_data.name)

  button.on_clicked = function()
    app_data.launch()
  end

  return button
end
```

### Timers and Async

```lua
-- Run after delay (milliseconds)
GLib.timeout_add(GLib.PRIORITY_DEFAULT, 1000, function()
  print("1 second elapsed")
  return false  -- Don't repeat
end)

-- Repeat every interval (seconds)
GLib.timeout_add_seconds(GLib.PRIORITY_DEFAULT, 5, function()
  print("5 seconds elapsed")
  return true  -- Keep repeating
end)

-- Run on next idle (main thread)
GLib.idle_add(GLib.PRIORITY_DEFAULT_IDLE, function()
  label:set_text("Updated from background thread")
  return false
end)

-- Spawn async process
GLib.spawn_async(nil,
  {'curl', '-s', 'https://example.com'},
  nil,
  GLib.SpawnFlags.SEARCH_PATH,
  nil,
  function(pid, stdout, stderr, status)
    if status == 0 then
      print("Output:", stdout)
    end
  end
)

-- Spawn and get output (blocking)
local ok, stdout, stderr, status = GLib.spawn_command_line_sync("uptime")
if ok then
  print(stdout)
end
```

## GdkPixbuf Image Manipulation

```lua
local GdkPixbuf = lgi.require("GdkPixbuf", "2.0")

-- Load image
local pixbuf = GdkPixbuf.Pixbuf.new_from_file("/path/to/image.jpg")

-- Load with scaling
local pixbuf = GdkPixbuf.Pixbuf.new_from_file_at_scale(
  "/path/to/image.jpg",
  width,   -- target width
  height,  -- target height
  true     -- preserve aspect ratio
)

-- Scale after loading
local scaled = pixbuf:scale_simple(width, height, 'BILINEAR')
-- Interpolation: NEAREST, TILES, BILINEAR, HYPER

-- Get dimensions
local width = pixbuf:get_width()
local height = pixbuf:get_height()

-- Access pixel data
local bytes = pixbuf:read_pixel_bytes()
local data = bytes:get_data()
local r = string.byte(data, 1)
local g = string.byte(data, 2)
local b = string.byte(data, 3)
```

## Drawing with Cairo

```lua
local cairo = lgi.require('cairo')

local drawing_area = Gtk.DrawingArea.new()
drawing_area:set_size_request(200, 200)

local function draw_func(area, cr, width, height)
  -- Set color (R, G, B, A)
  cr:set_source_rgba(1.0, 0.0, 0.0, 1.0)
  
  -- Draw rectangle
  cr:rectangle(10, 10, 100, 50)
  cr:fill()
  
  -- Draw circle
  cr:set_source_rgb(0.0, 0.0, 1.0)
  cr:arc(100, 100, 50, 0, 2 * math.pi)
  cr:stroke()
  
  -- Draw line
  cr:set_line_width(3)
  cr:set_line_cap(cairo.LineCap.ROUND)
  cr:move_to(0, 0)
  cr:line_to(width, height)
  cr:stroke()
  
  -- Draw text
  cr:set_source_rgb(0, 0, 0)
  cr:select_font_face("Sans", cairo.FontSlant.NORMAL, cairo.FontWeight.BOLD)
  cr:set_font_size(20)
  cr:move_to(10, 30)
  cr:show_text("Hello Cairo!")
end

drawing_area:set_draw_func(draw_func, nil, nil)

-- Trigger redraw
drawing_area:queue_draw()
```

## JSON Parsing

```lua
local Json = lgi.require("Json")

-- Parse JSON string
local json_str = '{"name": "CandyShell", "version": 1}'
local parser = Json.Parser.new()
local ok, err = parser:load_from_data(json_str, -1)

if ok then
  local root = parser:get_root():get_object()
  
  -- Get string member
  local name = root:get_string_member("name")
  
  -- Get number member
  local version = root:get_int_member("version")
  
  -- Get nested object
  local colors = root:get_object_member("colors")
  
  -- Get array
  local items = root:get_array_member("items")
  local length = items:get_length()
  
  -- Iterate array
  for i = 0, length - 1 do
    local element = items:get_element(i)
    print(element:get_string())
  end
else
  print("JSON parse error:", err)
end
```

### Performance

```lua
-- Cache expensive computations
local icon_texture = nil
local function get_icon_texture()
  if not icon_texture then
    local icon_buf = GdkPixbuf.Pixbuf.new_from_file_at_scale('res/icon.jpg', 100, 100)
    icon_texture = Gdk.Texture.new_for_pixbuf(icon_buf)

end
```

## Common Issues

<Warning>
**Issue**: Widgets not updating from background threads

**Solution**: Always update GTK widgets from the main thread using `GLib.idle_add()`:

```lua
-- Wrong
some_async_function(function(result)
  label:set_text(result)  -- Crashes!
end)

-- Correct
some_async_function(function(result)
  GLib.idle_add(GLib.PRIORITY_DEFAULT_IDLE, function()
    label:set_text(result)
    return false
  end)
end)
```
</Warning>

<Warning>
**Issue**: Memory leaks with pixbufs

**Solution**: Always clear references and run GC:

```lua
local pixbuf = GdkPixbuf.Pixbuf.new_from_file("large.jpg")
image:set_from_pixbuf(pixbuf)
pixbuf = nil  -- Clear reference
collectgarbage("step")
```
</Warning>

<Warning>
**Issue**: Timer callbacks returning `nil` instead of `true`/`false`

**Solution**: Always explicitly return a boolean:

```lua
-- Wrong - timer stops after first call
GLib.timeout_add_seconds(GLib.PRIORITY_DEFAULT, 5, function()
  update_widget()
  -- No return value = nil = false
end)

-- Correct
GLib.timeout_add_seconds(GLib.PRIORITY_DEFAULT, 5, function()
  update_widget()
  return true  -- Keep repeating
end)
```
</Warning>
